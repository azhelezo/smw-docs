# Внутренний документооборот
Пользователи могут создавать заявки на закупку материалов / оплату счетов поставщикам, подписывать или отклонять заявки в зависимости от уровня доступа. Уровень доступа устанавливает администратор системы через административную панель django.

По умолчанию установлено 4 уровня доступа:
- Руководитель отдела
- Менеджер по закупкам
- Финансовый контролер
- Генеральный директор

*изменение этих уровней описано в блоке Установка*

Так же, по умолчанию, доступ к заявкам осуществляется по такой логике:
- Пользователь видит только свои заявки
- Руководитель отдела видит только заявки своего отдела
- Менеджер по закупкам, Финансовый контролер и Генеральный директор видят все заявки

*изменение зон видимости описано в блоке Установка*

На каждой заявке автоматически создаются поля с названиями, соответствующими уровням доступа. При добавлении подписи, в соответствующем поле фиксируется решение имя подписавшего.

Заявка считается одобренной только если стоят все подписи и отклоненной - если стоит хотя бы одна "отклоняющая" подпись.

Автор заявки может вносить в нее изменения пока на ней не стоит хотя бы одна подпись.

Заявку можно сформировать в версии для печати.

# Установка
Склонируйте репозиторий.

Если хотите изменить уровни доступа - внесите изменения в список `LEVEL` в `users/models.py`.
`LEVEL_VIEW_ALL` - список уровней с правом видеть все заявки.

Находясь в директории проекта, выполните команды:
```
python manage.py makemigrations
python manage.py migrate
python manage.py createsuperuser
python manage.py runserver
```

Проект запущен и доступен по адресу [localhost:8000](http://localhost:8000/).
